Week 5 Notes: Testing and Debugging

Strict Mode

- It tells errors in the code.
- Add to the first line of a JavaScript file: 'use strict';
- It can also be added inside a function.

Linting Tools

Like in CSS. Run in terminal: stylelint and add file name.

Feature Detection

- Determines browser support.
- This is done using an IF statement to check whether an object or method exists before trying to actually call the method.

Example:
if (window.holoDeck) {
virtualReality.activate(); }

Debugging in the Browser

- Alert() is similar to console.log().
- Use breakpoints. Also works typing “debugger” in the breakpoint desired.

Error Objects

- An error object is created when an exception occurs.

Example:
const error = new Error('Oops, something went wrong');
Or a Throw exception:
throw new Error('Something has gone badly wrong!');

Exception Handling

Try, Catch, and Finally

function imaginarySquareRoot(number) {
'use strict';
if (number < 0) {
throw new RangeError("You can't find the square root of
➥ negative numbers")
}
return Math.sqrt(number);
};

Tests

Test-driven Development

  • 1. Write tests (that initially fail)
  • 2. Write code to pass the tests
  • 3. Refactor the code
  • 4. Test refactored code
  • 5. Write more tests for new features

Jest

  • Install -g jest.
  • Create a file named ending in .test.js (squareRoot.test.js).
  • Add code.
  • Run in terminal jest -c {}
